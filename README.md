# home-macaujc-journal

该仓库用于归档和发布多个独立的 HTML 页面。这些页面不针对任何单一域名或具体网站，而是作为通用文档与静态页面的存储与展示空间。

## 项目简介

本仓库的主要目的是以结构化的方式保存、组织并公开访问一组 HTML 文件。每个页面可独立查看，无需依赖特定后端服务。适用于长期归档、临时发布或静态内容展示场景。

## 目录说明

```
.
├── index.html          # 仓库首页 / 导航页（如有）
├── pages/              # 存放主要 HTML 页面
│   ├── example1.html
│   └── example2.html
├── assets/             # 静态资源（CSS、JS、图片等）
│   ├── style.css
│   └── images/
├── README.md           # 本说明文件
└── .gitignore          # Git 忽略规则
```

- `pages/`：所有独立 HTML 页面存放于此，可包含子目录。
- `assets/`：与页面关联的样式、脚本、媒体等资源文件。
- `index.html`：可选，用于提供页面列表或说明。

## 页面归档说明

- 每个 HTML 文件均为自包含页面（或依赖 `assets/` 下的资源）。
- 文件名应具有描述性，避免使用无意义缩写。
- 页面内容可包含文字、图片、链接等常见 HTML 元素。
- 不强制要求统一模板，但建议保持基本的 HTML 结构（`<!DOCTYPE html>`、`<head>`、`<body>` 等）。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并确保所有引用资源路径正确（建议使用相对路径）。
- 修改页面：直接编辑对应文件，提交后自动更新。
- 删除页面：移除文件并清理相关引用（如有）。
- 建议为重要变更添加简短的提交说明，便于追溯历史。
